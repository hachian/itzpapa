# Implementation Plan

## Tasks

- [x] 1. 型定義とi18nモジュールの基盤構築
- [x] 1.1 言語型の定義と設定インターフェースの拡張
  - サポート言語（日本語/英語）のリテラル型を定義する
  - 既存のサイト情報インターフェースに言語プロパティを追加する（オプショナル、デフォルトは日本語）
  - 後方互換性を維持し、未設定時は現在の日本語表示を維持する
  - _Requirements: 1.1, 1.2, 1.3, 5.1, 5.2, 5.3, 7.1_

- [x] 1.2 (P) 翻訳リソースファイルの作成
  - 翻訳テキストを管理する専用モジュールを作成する
  - ナビゲーション、目次、タグ、メニュー操作、フッターの全テキストを日英両言語で定義する
  - 翻訳キーの型を定義し、新規キー追加時に全言語のエントリを強制する
  - _Requirements: 6.1, 6.2, 6.3_

- [x] 1.3 翻訳ヘルパー関数の実装
  - 現在の言語設定を取得する関数を実装する
  - 翻訳キーからテキストを取得するヘルパー関数を実装する
  - 言語未設定時はデフォルトで日本語を返す
  - _Requirements: 4.1, 5.3, 7.1_

- [x] 2. UIコンポーネントの国際化対応
- [x] 2.1 ヘッダーコンポーネントのメニュー操作テキスト国際化
  - メニュー開閉ボタンのaria-labelを翻訳テキストに置き換える
  - JavaScript側のメニュー操作メッセージも翻訳対応する
  - _Requirements: 4.1_

- [x] 2.2 (P) フッターコンポーネントの著作権テキスト国際化
  - 著作権表示のデフォルトテキストを翻訳対応する
  - ユーザー定義のカスタムテキストがある場合は翻訳より優先する
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 2.3 (P) 目次コンポーネントのテキスト国際化
  - 目次タイトルとaria-labelを翻訳テキストに置き換える
  - 目次の表示/非表示トグルのaria-labelを翻訳対応する
  - CSSのcontentプロパティで設定されたテキストへの対応を行う
  - _Requirements: 4.1_

- [x] 2.4 (P) タグページのテキスト国際化
  - タグ一覧ページのタイトルと見出しを翻訳対応する
  - 個別タグページの「タグ一覧に戻る」リンクを翻訳対応する
  - タグリストのaria-labelを翻訳対応する
  - _Requirements: 4.1_

- [x] 3. 統合と動作確認
- [x] 3.1 設定ファイルへの言語プロパティ追加例の提供
  - 設定ファイルのコメントに言語オプションの説明を追加する
  - デフォルト設定を日本語として維持する
  - _Requirements: 1.1, 1.2, 1.3, 7.2, 7.3_

- [x] 3.2 ビルド確認と後方互換性テスト
  - 言語未設定の既存設定でビルドが成功することを確認する
  - 日本語設定での全コンポーネント表示を確認する
  - 英語設定での全コンポーネント表示を確認する
  - 日付表示形式がYYYY/MM/DDで維持されることを確認する
  - _Requirements: 4.2, 7.1, 7.2, 7.3_

- [x]* 3.3 翻訳ヘルパーの単体テスト
  - デフォルト言語取得の動作確認テストを作成する
  - 翻訳キー取得の動作確認テストを作成する
  - 全翻訳キーの存在確認テストを作成する
  - _Requirements: 5.3, 6.3_
