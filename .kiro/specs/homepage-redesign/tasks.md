# Implementation Plan

## Tasks

- [ ] 1. ヒーローセクションのデザイン刷新
- [x] 1.1 フェードインアニメーションの実装
  - ページロード時にヒーローセクション全体がフェードインするアニメーションを追加
  - 既存のdesign-tokensのduration/easing変数を参照してアニメーションを定義
  - アニメーション開始をLCP測定後に遅延させ、パフォーマンスへの影響を最小化
  - prefers-reduced-motion設定時にアニメーションが無効化されることを確認
  - _Requirements: 1.1, 1.2, 5.7_

- [x] 1.2 ロゴ画像のホバーインタラクション強化
  - ロゴ画像にホバー時のスケールアップ効果を追加
  - トランジションにdesign-tokensの変数を使用
  - ダークモードでも視覚的効果が適切に機能することを確認
  - _Requirements: 1.3, 2.4_

- [x] 1.3 モバイルレスポンシブレイアウトの調整
  - 768px未満で縦方向レイアウトに切り替わる既存の動作を確認
  - テキストの中央揃えとスペーシングの最適化
  - 320px〜480pxの極小画面でのフォントサイズ調整
  - _Requirements: 1.4, 6.1, 6.3_

- [ ] 2. ナビゲーションカードのデザイン統一
- [ ] 2.1 card.cssスタイルの適用と調整
  - 既存のcard.cssのホバー効果（リフト、シャドウ変化）が適用されていることを確認
  - フォーカスインジケーターの視認性を検証
  - カードアイコンのサイズと色をデザイントークンに統一
  - _Requirements: 2.1, 2.2, 2.3_

- [ ] 2.2 ダークモード対応の検証と調整
  - html.dark時のカード背景、ボーダー、テキストカラーを確認
  - コントラスト比が4.5:1以上であることを検証
  - 必要に応じてダークモード固有のスタイル調整を追加
  - _Requirements: 2.4, 5.6_

- [ ] 3. 最新記事セクションのデザイン強化
- [ ] 3.1 記事カードのスタイル統一
  - card.cssの共通スタイルを記事カードに適用
  - サムネイル、タイトル、説明文、日付の視覚的区分を明確化
  - ホバー時のリフトアップとシャドウ変化アニメーションを確認
  - _Requirements: 3.1, 3.2_

- [ ] 3.2 グリッドレイアウトのレスポンシブ対応
  - 画面幅に応じた1列〜3列のグリッド自動調整を確認
  - 各ブレークポイント（480px, 768px, 1024px）での表示を検証
  - カード間のギャップとパディングの最適化
  - _Requirements: 3.3, 6.4_

- [ ] 3.3 サムネイルなし記事の表示対応
  - サムネイル画像がない場合のカードレイアウトを調整
  - コンテンツ領域が適切に拡張されることを確認
  - 他のカードとの視覚的一貫性を維持
  - _Requirements: 3.4_

- [ ] 4. パフォーマンス最適化と検証
- [ ] 4.1 画像最適化の確認と調整
  - ヒーロー画像にloading="eager"とfetchpriority="high"が設定されていることを確認
  - 記事サムネイルにloading="lazy"が設定されていることを確認
  - すべての画像に適切なwidth/height属性があることを検証
  - _Requirements: 4.2, 4.3_

- [ ] 4.2 (P) Lighthouseパフォーマンス監査の実行
  - Lighthouse CLIまたはDevToolsでパフォーマンススコアを計測
  - LCPが2.5秒以内であることを確認
  - スコア90以上を達成していない場合は改善点を特定
  - _Requirements: 4.1, 4.5_

- [ ] 5. アクセシビリティ検証と改善
- [ ] 5.1 (P) キーボードナビゲーションの検証
  - すべてのインタラクティブ要素がTabキーでアクセス可能であることを確認
  - フォーカス順序が論理的であることを検証
  - フォーカスインジケーターが視認可能であることを確認
  - _Requirements: 5.2, 5.5_

- [ ] 5.2 (P) セマンティックHTMLとARIA属性の検証
  - 見出し構造（h1→h2→h3）が適切であることを確認
  - すべての画像に意味のあるalt属性があることを検証
  - ARIA属性（aria-labelledby, role, aria-label）が適切に使用されていることを確認
  - _Requirements: 5.3, 5.4_

- [ ] 5.3 (P) カラーコントラストの検証
  - テキストと背景のコントラスト比が4.5:1以上であることを確認
  - ライトモードとダークモードの両方で検証
  - 必要に応じてdesign-tokensのカラー値を調整
  - _Requirements: 5.6_

- [ ] 5.4 Lighthouseアクセシビリティ監査の実行
  - Lighthouse Accessibilityスコア90以上を確認
  - WCAG 2.1 AA基準への準拠を検証
  - 検出された問題があれば修正
  - _Requirements: 5.1_

- [ ] 6. レスポンシブデザインの総合検証
- [ ] 6.1 全ブレークポイントでの表示確認
  - 320px, 480px, 768px, 1024px, 1920pxでのレイアウト検証
  - 各セクション（ヒーロー、ナビカード、記事カード）の表示を確認
  - 横スクロールが発生しないことを検証
  - _Requirements: 6.1_

- [ ] 6.2 タッチターゲットサイズの検証
  - すべてのクリック可能要素が44x44px以上であることを確認
  - モバイルデバイスでのタップ操作をシミュレート
  - 必要に応じてパディングやサイズを調整
  - _Requirements: 6.2_

- [ ] 6.3 クロスブラウザ検証
  - Chrome, Firefox, Safari, Edgeでの表示確認
  - iOS Safari, Android Chromeでのモバイル表示確認
  - アニメーションとトランジションの動作検証
  - _Requirements: 6.1_
