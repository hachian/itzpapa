# Implementation Plan

## Tasks

- [x] 1. テーブルスタイルの基盤構築
- [x] 1.1 CSS変数とファイル構成のセットアップ
  - table.css ファイルを新規作成し、テーブル専用のCSS変数を定義
  - ライトモード用変数: 境界線色、ヘッダー背景色、偶数行背景色、ホバー背景色
  - design-tokens.cssで定義済みの変数を参照し、一貫性を保つ
  - global.cssに@importを追加してtable.cssを読み込む
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 1.2 基本テーブルスタイルの実装
  - テーブル要素の幅、ボーダーコラプス、フォントサイズを設定
  - ヘッダーセル（th）に背景色、フォントウェイト、パディング、下部境界線を適用
  - ボディセル（td）にパディング、下部境界線、最小幅を適用
  - border-collapse: separateとborder-spacing: 0を使用して角丸対応の基盤を作る
  - _Requirements: 1.1, 1.2, 1.3, 3.3_

- [x] 2. テーブル角丸と可読性向上
- [x] 2.1 テーブル角丸の実装
  - テーブル全体にborder-radiusを適用
  - 最初の行の最初と最後のセル、最後の行の最初と最後のセルに個別の角丸を適用
  - overflow: hiddenで角丸からのはみ出しを防止
  - _Requirements: 1.4, 5.3_

- [x] 2.2 ゼブラストライプとホバーエフェクトの実装
  - 偶数行に背景色を適用（nth-child(even)セレクタを使用）
  - 行ホバー時に背景色を変更するトランジションを追加
  - ホバー色にはプライマリカラーの透明度バリエーションを使用
  - _Requirements: 4.1, 4.2_

- [x] 3. ダークモード対応
- [x] 3.1 ダークモード用CSS変数の定義
  - html.darkセレクタでダークモード用の変数値をオーバーライド
  - 境界線色、ヘッダー背景色、偶数行背景色、ホバー色をダークモード用に調整
  - design-tokens.cssのダークモードカラーパレットを参照
  - _Requirements: 2.1, 2.2_

- [x] 3.2 OSダークモード設定対応とトランジション
  - @media (prefers-color-scheme: dark)でOS設定に基づくダークモード対応
  - :root:not(.light)セレクタを使用して明示的ライトモード以外をカバー
  - 背景色とボーダー色の変更にトランジションを適用
  - _Requirements: 2.1, 2.3_

- [x] 4. レスポンシブ対応
- [x] 4.1 水平スクロールの実装
  - prose内のテーブルにoverflow-x: autoを適用
  - テーブルコンテナにマージンとシャドウを追加
  - スクロール時のコンテンツ切れを防ぐための最小幅を確保
  - _Requirements: 3.1, 3.3_

- [x] 4.2 モバイル対応とスクロールインジケーター
  - モバイルブレイクポイント（767px以下）でパディングとフォントサイズを調整
  - スクロール可能な状態を示す影グラデーションをテーブル端に適用
  - タッチデバイスでのスクロール体験を考慮
  - _Requirements: 3.2_

- [x] 4.3 テキスト配置サポート
  - align属性に基づくテキスト配置スタイルを定義
  - 左揃え、中央揃え、右揃えのサポート
  - Markdownテーブルの配置構文との互換性を確保
  - _Requirements: 4.3_

- [x] 5. 統合と視覚テスト
- [x] 5.1 統合確認と視覚テスト
  - ビルドを実行し、CSSが正しくインポートされることを確認
  - 既存ブログ記事のテーブルでスタイルが適用されることを視覚的に検証
  - ライトモード、ダークモード、モバイル幅での表示を確認
  - ゼブラストライプとホバーエフェクトの動作を確認
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 3.1, 3.2, 3.3, 4.1, 4.2, 4.3, 5.1, 5.2, 5.3, 5.4_
