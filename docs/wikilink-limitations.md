# Wikilink Plugin 制限事項と回避策

## テーブル内でのWikilink使用制限

### 問題
Markdownテーブル内でエイリアス付きWikilinkを使用する際、パイプ文字（`|`）がテーブルの区切り文字として解釈されるため、正しく動作しません。

### 影響を受ける記法
```markdown
| リンク |
|--------|
| [[../page/index.md|エイリアス]] |  ← 動作しない
```

### 回避策

#### 1. エイリアスなしのWikilinkを使用
```markdown
| リンク |
|--------|
| [[../page/index.md]] |  ← 動作する
```

#### 2. 通常のMarkdownリンクを使用
```markdown
| リンク |
|--------|
| [エイリアス](/blog/page) |  ← 動作する
```

#### 3. テーブル外でリンクを定義
```markdown
テーブル外: [[../page/index.md|エイリアス]]

| 説明 | 詳細 |
|------|------|
| リンク | 上記参照 |
```

## 技術的背景

この制限は、Markdown処理の順序に起因します：

1. **GFM（GitHub Flavored Markdown）** がテーブルを最初に解析
2. パイプ文字でセルが分割される
3. Wikilinkプラグインが動作する時点では、すでにWikilinkが壊れている

## 将来の改善案

1. **カスタムエスケープシーケンス**: `[[../page/index.md\|エイリアス]]`
2. **代替記法**: `[[../page/index.md::エイリアス]]`
3. **プリプロセッサー**: テーブル解析前にWikilinkを処理

現時点では、上記の回避策を使用することを推奨します。
