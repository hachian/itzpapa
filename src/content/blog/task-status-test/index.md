# タスクステータス機能のテスト

このページでは、Obsidian形式のタスクステータス記法（22種類の拡張チェックボックス）の動作をテストします。

## 1. 基本タスクステータス

基本的なタスクの状態を示すステータスです。

- [ ] `space` to-do
- [/] `/` incomplete
- [x] `x` done
- [-] `-` canceled

## 2. スケジューリング関連

タスクのスケジュール状態を示すステータスです。

- [>] `>` forwarded
- [<] `<` scheduling

## 3. 重要度・状態マーカー

タスクの重要度や特殊な状態を示すステータスです。

- [?] `?` question
- [!] `!` important
- [*] `*` star

## 4. 参照・情報マーカー

参照情報やメタデータを示すステータスです。

- ["] `"` quote (引用)
- [l] `l` location
- [b] `b` bookmark
- [i] `i` information

## 5. アイデア・評価マーカー

アイデアや評価を示すステータスです。

- [S] `S` savings
- [I] `I` idea
- [p] `p` pros
- [c] `c` cons

## 6. アクション・結果マーカー

アクションや結果を示すステータスです。

- [f] `f` fire
- [k] `k` key
- [w] `w` win
- [u] `u` up
- [d] `d` down

## 7. 複合テスト

### 7.1 ネストされたタスク

- [x] 親タスク（完了）
  - [/] 子タスク1（進行中）
  - [ ] 子タスク2（未完了）
  - [!] 子タスク3（重要）

### 7.2 複数のタスクを含むリスト

プロジェクトのタスク一覧：

- [x] 要件定義を完了する
- [x] 設計書を作成する
- [/] 実装を進める
- [ ] テストを実行する
- [>] レビューを次週に持ち越し
- [?] 仕様について確認が必要

### 7.3 タスクと通常テキストの混在

以下はタスクリストです：

- [!] 重要なタスクには`!`マークを使用
- [I] 良いアイデアは`I`でマーク
- [*] お気に入りは`*`でスター付け

通常のリストアイテム：
- これは通常のリストです
- タスク記法ではありません

## 8. エッジケース

### 8.1 未知のステータス文字

- [z] 未定義のステータス（unknownとして処理される）
- [9] 数字のステータス（unknownとして処理される）

### 8.2 無効な構文（通常リストとして表示）

- [ab] 複数文字（タスクとして処理されない）
- [] 空のブラケット（タスクとして処理されない）
- [x]テキスト（スペースなし、タスクとして処理されない）

### 8.3 コードブロック内（処理されない）

```markdown
- [x] これはコードブロック内のタスク
- [ ] 処理されずそのまま表示される
```

### 8.4 インラインコード内

インラインコードの例：`- [x] タスク` はそのまま表示されます。
